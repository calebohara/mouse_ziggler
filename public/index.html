How it works

Three small, polite things — every minute or so. Nothing visible on screen.

1

Tells Windows you're here

It quietly registers a tiny bit of activity — the same signal Teams checks to decide if you've stepped away.

2

Keeps the screen awake

Pins the display and the system on so you don't lose your work to a sleep timer mid-thought.

3

Checks itself

Reads back the same idle counter Teams uses. If something didn't take, it logs it instead of silently lying to you.

When to use it

Honest about where it helps and where it doesn't.

✓ Use noidle.app when…

  • You're at your desk reading or thinking and don't want Teams to mark you Away.
  • You want your machine to stay awake without changing Windows' power settings forever.
  • You take long phone calls away from the keyboard but stay in front of the laptop.
  • You want the same behaviour on Slack and any app that uses Windows' idle signal.

○ It can't help when…

  • Your laptop is locked (Win+L). Teams marks you Away by OS design — no software jiggler can override that.
  • Outlook says you're in a meeting. Calendar status wins.
  • You're on a managed work laptop with corporate monitoring software (CrowdStrike, Sentinel, Teramind, ActivTrak — they detect synthetic input). Don't run noidle.app there without permission.
  • You're on a disconnected Remote Desktop session.

Questions you'd ask a friend

If we missed yours, open an issue.

Is this a virus? My antivirus complained.

No. The exe is built with PyInstaller (a tool that bundles Python apps into a single file), and Windows Defender SmartScreen flags lots of PyInstaller exes by default — it's a heuristic against the wrapper format, not the contents. The first time you run it you'll see "Windows protected your PC." Click More info → Run anyway.

You can verify the file before running it by uploading it to VirusTotal, and the full source is open on GitHub.

Will my employer be able to tell I'm using it?

If your employer runs corporate endpoint monitoring (CrowdStrike, Sentinel, Teramind, ActivTrak, etc.), yes — those tools intentionally hook the Windows input pipeline and flag synthetic input. noidle.app makes no attempt to hide from them.

On a personal machine with no monitoring software, no — there's no telemetry, no network calls except the one update check (which you can turn off in the tray menu), and no special API that distinguishes "real" mouse movement from injected.

Does it work with Slack? Discord? Anything besides Teams?

Yes — anything that uses Windows' idle signal (which is most desktop apps that show a presence indicator) will see your machine as active. That includes Slack, Discord, the new Teams (WebView2), and most chat clients.

It does not influence presence in browser-only tools (e.g. Slack open in a Chrome tab) since browsers track tab visibility, not OS idle. For those, leave the tab focused.

How is this different from Caffeine / Mouse Jiggler?

Caffeine just prevents sleep — it doesn't stop Teams from going Away because it doesn't reset Windows' input idle counter. Old-school mouse jigglers move the cursor visibly, which is annoying mid-click and obvious on a screen-share.

noidle.app does both: prevents sleep and resets the idle counter, with a sub-pixel nudge that's invisible, plus auto-pause when you're typing or screen-sharing. And it's free, MIT, and doesn't phone home.

Is it free? What's the catch?

Free, MIT-licensed, no telemetry, no upsell. Built as a personal project. The only "catch" is you should not run it on a managed work device without your IT team's blessing.

Does it work on Mac or Linux?

Not yet — it relies on Win32 APIs (SendInput, SetThreadExecutionState, GetLastInputInfo) that don't exist on macOS or Linux. A cross-platform port would be most of a rewrite.

On macOS, the built-in caffeinate command in Terminal does a similar job. On Linux, look at caffeine or xdotool.

How do I uninstall it?

If you used the .msi: Settings → Apps → noidle.app → Uninstall.

If you used the portable .exe: right-click the tray icon → Quit, then delete the .exe. To also remove settings: delete %APPDATA%\noidle\ and %LOCALAPPDATA%\noidle\. If you toggled "Start with Windows," disable that first so the registry entry is cleaned up.
